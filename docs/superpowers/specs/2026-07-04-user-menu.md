# User Menu — Assessment + Implementation Plan

**Date:** 2026-07-04
**Status:** PLANNED — not executed. Execution slated for Sonnet.
**Origin:** Review finding #2 (node management via menu). Assessment by Fable, plan by Opus.
**Addresses:** the graph page has ZERO chrome and the app has NO logout anywhere — a user currently cannot sign out.

---

## Verified current state (checked in code 2026-07-04)

- `/graph` (`app/(app)/graph/page.tsx`) is a server component → renders `<GraphWrapper>` → `<GraphCanvas>` + `<ChatPanel>`. No header, no menu, no logout.
- **No `signOut` call exists anywhere in the app.**
- Middleware (`middleware.ts`) **already** bounces a signed-out user off `/graph` → `/login`, and a signed-in user off `/login|/signup` → `/graph`. So logout = `supabase.auth.signOut()` + `router.push("/login")`; middleware does the rest. No extra guard needed.
- Auth is client-side Supabase (`lib/supabase/client.ts`, `signInWithPassword`).
- Profile data that exists today (`profiles` table): `display_name`, `base_profile {age, home_location}`, `ad_preferences {category: bool}`. Only the extraction pipeline reads these — **no UI writes them yet**; this menu is the first writer.
- Graph nodes = `vault_notes` rows; edges = `vault_links`. `vault_links.source_note_id` / `target_note_id` are **`ON DELETE CASCADE`** (schema.sql:35–36) → deleting a note auto-removes its edges at the DB level. Only the in-memory graph needs a refresh nudge.
- `GraphWrapper` owns `refreshTrigger` (bumped on chat reply → canvas re-fetches). Menu deletes must bump the same trigger.
- `ChatPanel` overlay is `z-30`, `pointer-events-none` with `pointer-events-auto` only on the input strip. Chat owns top-center and bottom-center; **corners are free**.
- Root node is `_profile.md` / title "You" / topic "Profile" / `source="system"` — extraction hard-depends on it.

## Placement & interaction (Fable, kept)

- **Trigger:** ghosted **avatar chip** (circle, first initial of `display_name`) in the **top-right corner**. Low opacity until hover — preserves the zero-chrome aesthetic. Corners are free of chat.
- **Opens:** a **right-side slide-over drawer** (~360–400px), dark translucent glass, graph still visible behind it. Sub-views (Manage nodes) push a **second-level view inside the same drawer** with a back arrow — **no route changes, no separate pages**.
- Esc / click-outside closes. **Full-screen sheet on mobile widths.**
- Drawer must be `pointer-events-auto` and sit **above chat** (`z-40+`) so graph pan/zoom behind it is swallowed.

## Slice 2 — Manage nodes + Export (first week of real users)

**Manage nodes view** (inside `UserMenu`, second-level):
- Fetch via **browser client** (RLS-scoped, never admin): `vault_notes.select("id, title, entity_type, path, content_md, source, created_at")`, newest-first.
- **Hide `_profile.md`** entirely (root anchor).
- Find-in-list: entity-type **filter chips** (the 7 types) + **text search** box. No pagination until it hurts.
- Row tap → **read-only rendered markdown** (view before trusting delete).
- **Delete:** confirm dialog stating link count (`vault_links` where `source_note_id = id OR target_note_id = id`) → `vault_notes.delete().eq("id", id)` → cascade removes edges → **bump `refreshTrigger`** so the canvas drops the node.
- **Block delete of `source="system"` nodes** (brand hubs) — deleting one while items still link orphans structure (Fable risk #4). Simplest: no delete button on system rows.
- **Resurrection follow-up (investigate during slice 2):** deleting a node doesn't stop re-extraction — the agenda may re-create it next conversation. Minimum: clear matching agenda items on delete. Real fix (slice 3): a do-not-re-extract tombstone. Locate where the agenda persists before finalizing.

**Export** (new route `apps/web/app/api/vault/export/route.ts`, server):
- Server client (user session, RLS) → all `vault_notes` → zip by `path` (frontmatter + wikilinks intact = a real Obsidian vault) → return `.zip`.
- **Decision needed:** zip requires a dep (e.g. `jszip`). Lazy fallback = single concatenated `.md`, but multi-file zip is the on-brand answer. Pick at execution time.

## Cross-cutting rules

- **RLS user client only** for all node reads/deletes and export — never the admin client (cross-user data-loss risk).
- Cascade handles edge cleanup; **dangling `[[wikilinks]]` left inside other notes' markdown are acceptable** (Obsidian tolerates them) — don't rewrite content.
- Every delete must fire `refreshTrigger` or the node visibly won't disappear.
- Consistent dark/minimal aesthetic; reuse landing-page styling cues where sensible.
